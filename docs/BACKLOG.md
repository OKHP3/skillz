## Maturity model

Skills move through six levels. No skill is promoted without meeting the gate for its level.

| Level | Label | Gate |
|---:|---|---|
| 0 | Placeholder | Family or concept exists only — no SKILL.md content |
| 1 | Skeleton | SKILL.md shell and section structure exist |
| 2 | Draftable | Instructions are coherent but not fully tested |
| 3 | Usable | Can be used manually with acceptable output |
| 4 | Validated | Has worked examples, validation gates, and a review checklist |
| 5 | Published | Release-tagged, registry-ready, and public-facing |

Do not mark a skill above Level 1 until it has enough instruction content to be used without tribal knowledge.
Do not mark a skill Level 4 until at least one worked example and validation checklist exist.
Do not mark a skill Level 5 until the repo has a release tag and the publishing checklist in `PUBLISHING.md` is complete.

## Catalog upkeep

The skill catalog (`SKILLS.md`) is auto-generated by `okhp3-skill-cataloger --full`. Do not edit it by hand.

Update `AGENTS.md` whenever a skill is added, removed, renamed, or promoted to a new maturity level. Run the cataloger after any structural change to regenerate `SKILLS.md`.

Update this file (`BACKLOG.md`) when:
- A new planned family is added or promoted
- A skill reaches a new maturity level
- The promotion priority order changes
