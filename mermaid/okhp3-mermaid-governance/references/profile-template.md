## Semantic Class Library

The classes below are the only `classDef` names used in this project. New node roles not listed here must be added to this profile before being used — do not introduce undeclared class names.

| Class name | Semantic role | Notes |
|---|---|---|
| `process` | A work step — human or automated task | Primary diagram element |
| `decision` | A gateway — branching or merging logic | XOR/AND/OR gateway shapes |
| `external` | A system, actor, or entity outside the project boundary | Use for third-party systems, regulators, end users |
| `dataStore` | A persistent store — database, file, registry | Use cylinder or database node shape |
| `event` | A trigger or terminal event (start, end, timer, message) | Start/end events in BPMN-informed flows |
| `annotation` | Supplementary context outside the main flow | Non-flow explanatory nodes |

> To add a class: add a row here, increment the profile version, and re-declare.

The `classDef` block for these classes is produced by `okhp3-mermaid-theme-builder` when the palette is applied. Semantic names are stable; color values come from the palette and may change if the palette is updated.

## Conformance Expectations

A diagram is **Compliant** with this profile when all of the following are true:

1. The `%%{init}%%` block matches the palette declared above (hex values align with `assets/palettes.json` for the declared palette ID).
2. All `classDef` declarations match the semantic class library above.
3. Every node has a `class` assignment from the declared library. No node is unstyled.
4. The diagram passes all three gates from `okhp3-mermaid-core`: syntax, semantic, and audience fit.
5. No excluded diagram types are used.
6. Provenance is respected: if this profile is Private, diagram content does not appear in public-facing artifacts.
7. The diagram has an entry in `DIAGRAMS.md`.

A diagram is **Partially compliant** if 1–2 items fail. It is **Non-compliant** if 3 or more fail, or if items 1 or 6 fail (palette mismatch and provenance violation are always critical).

To check conformance: load `okhp3-mermaid-governance` in Check mode and provide this GOVERNANCE.md alongside the diagram.
