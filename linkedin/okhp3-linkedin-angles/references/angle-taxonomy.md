# Angle Taxonomy

Categories for triaging a finished artifact into LinkedIn-postable angles. The "Surprising / Contrarian" category is a MANDATORY minimum per candidate set — angle-mining that only produces Summary/Lesson angles has failed, even if those angles are individually fine.

| Category | What it looks like | Mandatory? |
|---|---|---|
| Summary / Lesson | "Here's what I learned doing X" — the safe, obvious angle | No — but usually present by default |
| Surprising / Contrarian | A finding that cuts against the reader's expectation, deflates hype, or reverses the obvious framing. Example from this repo's own origin story: "I almost spent a weekend competing in a skills marketplace where #1 has 116 downloads" | **Yes — at least one candidate per set** |
| Teardown / Critique | Structural critique of an existing tool, pattern, or piece of work — constructive, acknowledges real strengths, identifies gaps | No |
| Behind-the-Scenes / Process | "Here's how this thing was actually built" — the system-story angle, especially valuable when the process itself demonstrates a capability | No |
| Prediction / Trend | Forward-looking claim based on evidence in the source artifact | No |

## Why Surprising/Contrarian is mandatory

Angle-mining that defaults to safe summaries is the most likely failure mode for this skill — it's the path of least resistance and produces forgettable content. Forcing at least one contrarian candidate per set keeps the registry from filling up with interchangeable "lessons learned" posts.

## Sizing

Each candidate also gets a suggested length:
- **Post** — single insight, fits LinkedIn's consolidated-paragraph format, no subheadings
- **Article-worthy** — needs more room than a post allows; flag for the (not yet scaffolded) articles skill
