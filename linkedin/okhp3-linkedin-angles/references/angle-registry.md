# Angle Registry - Template

Candidates land here. This is the angle backlog - capture now, triage later, but in a structured place.

| Date | Source artifact | Category | One-line summary | Suggested length | BFS flag | Status |
|---|---|---|---|---|---|---|
| YYYY-MM-DD | Filename or "conversation: [topic]" | Summary/Lesson / Surprising-Contrarian / Teardown / Behind-the-Scenes / Prediction | One sentence | post / article | yes/no | new / drafting / posted / archived |

## Optional source packet note

When an angle depends on public X evidence, add a short note below the row:

```text
Source packet: query="<query>", window="<date range>", sample=<n>, collected_at="<ISO time>", sources="<URLs or post IDs>", limits="<sampling limits>"
```

Use a source packet only when it materially improves the angle. Keep copied content out of the registry; store durable references and a qualitative pattern summary instead.

## Status definitions

- **new** - candidate identified, not yet drafted
- **drafting** - `okhp3-linkedin-post` is working on it
- **posted** - published, link optional in a notes column if this template gets extended
- **archived** - decided against, kept for record (don't silently delete - useful to see what got passed over)
